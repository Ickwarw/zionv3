1. VISÃO GERAL DO SISTEMA
================================================================================

O Sistema de Dashboard é uma plataforma de business intelligence (BI) que 
permite criar visualizações interativas de dados para análise de leads e 
métricas de negócio.

PRINCIPAIS FUNCIONALIDADES:
✓ Criação de dashboards personalizados
✓ 220+ assuntos de análise disponíveis (1 por coluna da tabela lead)
✓ 6 tipos de gráficos (bar, line, pie, doughnut, radar, polarArea)
✓ Combinação de múltiplos assuntos em um único gráfico
✓ Salvamento de configurações de gráficos
✓ 12 gráficos predefinidos prontos para uso
✓ Formato compatível com Chart.js
✓ API REST completa para integração
✓ Atualização automática de dados
✓ Estatísticas gerais do sistema

USUÁRIOS DO SISTEMA:
- Gerentes e Diretores (visão estratégica)
- Analistas de BI (análise profunda)
- Equipe de Vendas (métricas de performance)
- Equipe de Pós-Venda (satisfação e retenção)
- Administradores (monitoramento geral)

CATEGORIAS DE ANÁLISE:
📊 Informações Básicas: Análise de razão social, fantasia, tipo pessoa
📞 Contatos: Distribuição de canais de contato (email, telefone, celular)
📍 Endereços: Análise geográfica (cidade, UF, bairro, região)
🎯 Classificação: Prioridades, status, departamentos, scores
💰 Financeiro: Status de débito, valores, formas de pagamento
📅 Temporalidade: Análise de datas de cadastro, ativação, cancelamento
🔄 Atividade: Chamados, interações, histórico
📈 Performance: Conversões, taxas, funil de vendas
😊 Satisfação: NPS, CSAT, contentamento
🌐 Origem: Canais de captação, campanhas, mídias

3. BANCO DE DADOS - ESTRUTURA COMPLETA
================================================================================

TABELA: dashboard_graficos
Localização: PostgreSQL zioncrm.dashboard_graficos

Propósito: Armazenar configurações de gráficos personalizados criados pelos 
           usuários

ESTRUTURA:

CREATE TABLE dashboard_graficos (
    id                  SERIAL PRIMARY KEY,
    titulo              VARCHAR(200) NOT NULL,
    descricao           TEXT,
    tipo_grafico        VARCHAR(50) NOT NULL,
    assuntos            JSONB NOT NULL,
    configuracoes       JSONB,
    data_criacao        TIMESTAMP DEFAULT NOW(),
    data_atualizacao    TIMESTAMP DEFAULT NOW(),
    ativo               BOOLEAN DEFAULT TRUE
);

COLUNAS DETALHADAS:

id (SERIAL PRIMARY KEY)
    - Identificador único do gráfico
    - Auto-incremento
    - Usado nas referências da API

titulo (VARCHAR(200) NOT NULL)
    - Título do gráfico
    - Exibido no topo da visualização
    - Obrigatório
    - Exemplo: "Leads por Cidade - São Paulo"

descricao (TEXT)
    - Descrição detalhada do gráfico
    - Opcional
    - Ajuda outros usuários a entender o propósito
    - Exemplo: "Análise das leads da cidade de São Paulo por prioridade"

tipo_grafico (VARCHAR(50) NOT NULL)
    - Tipo de visualização
    - Obrigatório
    - Valores aceitos:
        * "bar" - Gráfico de barras verticais
        * "line" - Gráfico de linhas
        * "pie" - Gráfico de pizza
        * "doughnut" - Gráfico de rosca
        * "radar" - Gráfico radar/aranha
        * "polarArea" - Gráfico de área polar
    - Constraint: CHECK (tipo_grafico IN (...))

assuntos (JSONB NOT NULL)
    - Array JSON com lista de assuntos a exibir
    - Obrigatório (mínimo 1 assunto)
    - Formato: ["prioridade", "status_vendas", "cidade"]
    - Cada assunto corresponde a uma coluna da tabela lead
    - Permite análise combinada de múltiplas dimensões
    - Exemplo: ["distribuicao_prioridade", "distribuicao_cidade"]

configuracoes (JSONB)
    - Configurações adicionais do gráfico
    - Opcional
    - Estrutura:
        {
            "colors": ["#FF6384", "#36A2EB", "#FFCE56"],
            "showLegend": true,
            "animation": true,
            "responsive": true,
            "maintainAspectRatio": false,
            "scales": {...},
            "plugins": {...}
        }
    - Compatível com opções do Chart.js

data_criacao (TIMESTAMP DEFAULT NOW())
    - Data e hora de criação do gráfico
    - Preenchido automaticamente
    - Não é alterado

data_atualizacao (TIMESTAMP DEFAULT NOW())
    - Data e hora da última modificação
    - Atualizado automaticamente via trigger
    - Usado para cache e sincronização

ativo (BOOLEAN DEFAULT TRUE)
    - Status do gráfico
    - TRUE = ativo (exibir)
    - FALSE = inativo/arquivado (soft delete)
    - Permite desativar sem deletar dados

ÍNDICES:

CREATE INDEX idx_dashboard_graficos_ativo
    ON dashboard_graficos(ativo);
    -- Acelera busca de gráficos ativos

CREATE INDEX idx_dashboard_graficos_tipo
    ON dashboard_graficos(tipo_grafico);
    -- Filtra por tipo de gráfico

CREATE INDEX idx_dashboard_graficos_data_criacao
    ON dashboard_graficos(data_criacao DESC);
    -- Ordena por mais recentes

CREATE INDEX idx_dashboard_graficos_assuntos
    ON dashboard_graficos USING GIN(assuntos);
    -- Busca por assuntos específicos (GIN index para JSONB)

TRIGGER:

CREATE OR REPLACE FUNCTION update_dashboard_graficos_timestamp()
RETURNS TRIGGER AS $$
BEGIN
    NEW.data_atualizacao = NOW();
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_update_dashboard_graficos_timestamp
    BEFORE UPDATE ON dashboard_graficos
    FOR EACH ROW
    EXECUTE FUNCTION update_dashboard_graficos_timestamp();

Propósito: Atualiza automaticamente data_atualizacao em cada UPDATE

CONSTRAINTS:

ALTER TABLE dashboard_graficos
    ADD CONSTRAINT check_tipo_grafico
    CHECK (tipo_grafico IN ('bar', 'line', 'pie', 'doughnut', 'radar', 'polarArea'));

ALTER TABLE dashboard_graficos
    ADD CONSTRAINT check_assuntos_not_empty
    CHECK (jsonb_array_length(assuntos) > 0);

4. RENDERIZAR GRÁFICO
    let chartInstance = null;
    
    async function renderizarGrafico(id) {
        const response = await fetch(`http://localhost:5004/api/dashboard/graficos/${id}/dados`);
        const data = await response.json();
        
        document.getElementById('chart-title').textContent = data.grafico.titulo;
        
        // Limpar gráfico anterior
        if (chartInstance) {
            chartInstance.destroy();
        }
        
        // Combinar datasets de todos os assuntos
        const labels = data.data[0].chartData.labels;
        const datasets = data.data.map((assunto, index) => ({
            label: assunto.label,
            data: assunto.chartData.datasets[0].data,
            backgroundColor: assunto.chartData.datasets[0].backgroundColor,
            borderColor: '#ffffff',
            borderWidth: 2
        }));
        
        // Criar novo gráfico
        const ctx = document.getElementById('main-chart').getContext('2d');
        chartInstance = new Chart(ctx, {
            type: data.grafico.tipo_grafico,
            data: {
                labels: labels,
                datasets: datasets
            },
            options: {
                responsive: true,
                maintainAspectRatio: false,
                plugins: {
                    legend: {
                        display: true,
                        position: 'top'
                    },
                    title: {
                        display: true,
                        text: data.grafico.titulo
                    }
                }
            }
        });
    }

5. CRIAR NOVO GRÁFICO
    async function salvarNovoGrafico(event) {
        event.preventDefault();
        
        const titulo = document.getElementById('input-titulo').value;
        const descricao = document.getElementById('input-descricao').value;
        const tipo_grafico = document.getElementById('input-tipo').value;
        
        const assuntosSelecionados = [];
        document.querySelectorAll('#assuntos-checkboxes input:checked').forEach(cb => {
            assuntosSelecionados.push(cb.value);
        });
        
        const response = await fetch('http://localhost:5004/api/dashboard/graficos', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json'
            },
            body: JSON.stringify({
                titulo: titulo,
                descricao: descricao,
                tipo_grafico: tipo_grafico,
                assuntos: assuntosSelecionados
            })
        });
        
        const data = await response.json();
        
        if (data.success) {
            alert('Gráfico criado com sucesso!');
            fecharModal();
            await carregarGraficosSalvos();
        } else {
            alert('Erro ao criar gráfico: ' + data.error);
        }
    }

6. CARREGAR ASSUNTOS DISPONÍVEIS
    async function carregarAssuntosDisponiveis() {
        const response = await fetch('http://localhost:5004/api/dashboard/assuntos');
        const data = await response.json();
        
        const container = document.getElementById('assuntos-checkboxes');
        container.innerHTML = '';
        
        // Agrupar por categoria
        const categorias = {};
        data.data.forEach(assunto => {
            if (!categorias[assunto.categoria]) {
                categorias[assunto.categoria] = [];
            }
            categorias[assunto.categoria].push(assunto);
        });
        
        // Renderizar por categoria
        Object.keys(categorias).forEach(categoria => {
            const section = document.createElement('div');
            section.className = 'categoria-section';
            section.innerHTML = `<h4>${categoria}</h4>`;
            
            categorias[categoria].forEach(assunto => {
                const checkbox = document.createElement('label');
                checkbox.innerHTML = `
                    <input type="checkbox" value="${assunto.key}">
                    ${assunto.label}
                    <small>${assunto.descricao}</small>
                `;
                section.appendChild(checkbox);
            });
            
            container.appendChild(section);
        });
    }

    async function deletarGrafico(id) {
        if (!confirm('Tem certeza que deseja deletar este gráfico?')) {

        }
        
        const response = await fetch(
            `http://localhost:5004/api/dashboard/graficos/${id}`,
            { method: 'DELETE' }
        );
        
        const data = await response.json();
        
        if (data.success) {
            alert('Gráfico deletado!');
            await carregarGraficosSalvos();
        }
    }

8. TIPOS DE GRÁFICOS E CHART.JS
================================================================================

O sistema suporta 6 tipos de gráficos do Chart.js:

--- TIPO 1: BAR (Barras Verticais) ---

Uso: Comparar categorias discretas
Ideal para: Contagens, distribuições, comparações

Exemplo:
new Chart(ctx, {
    type: 'bar',
    data: {
        labels: ['NovaLead', 'Fria', 'Quente'],
        datasets: [{
            label: 'Número de Leads',
            data: [50, 120, 150],
            backgroundColor: ['#dcfce7', '#dbeafe', '#fb923c']
        }]
    },
    options: {
        responsive: true,
        scales: {
            y: {
                beginAtZero: true
            }
        }
    }
});

--- TIPO 2: LINE (Linhas) ---

Uso: Tendências ao longo do tempo
Ideal para: Séries temporais, evolução, crescimento

Exemplo:
new Chart(ctx, {
    type: 'line',
    data: {
        labels: ['Jan', 'Fev', 'Mar', 'Abr'],
        datasets: [{
            label: 'Cadastros por Mês',
            data: [50, 75, 90, 120],
            borderColor: '#3B82F6',
            backgroundColor: 'rgba(59, 130, 246, 0.1)',
            tension: 0.4
        }]
    },
    options: {
        responsive: true
    }
});

--- TIPO 3: PIE (Pizza) ---

Uso: Proporções e percentuais
Ideal para: Distribuições, composições, partes do todo

Exemplo:
new Chart(ctx, {
    type: 'pie',
    data: {
        labels: ['Vendas', 'Pós-Venda'],
        datasets: [{
            data: [300, 200],
            backgroundColor: ['#3B82F6', '#10B981']
        }]
    },
    options: {
        responsive: true,
        plugins: {
            legend: {
                position: 'right'
            }
        }
    }
});

--- TIPO 4: DOUGHNUT (Rosca) ---

Uso: Similar ao PIE, mas com buraco no centro
Ideal para: Mesmos casos do PIE, visualmente mais moderno

Exemplo:
new Chart(ctx, {
    type: 'doughnut',
    data: {
        labels: ['Em Dia', 'Em Atraso'],
        datasets: [{
            data: [450, 50],
            backgroundColor: ['#10B981', '#EF4444']
        }]
    },
    options: {
        responsive: true,
        cutout: '70%'  // Tamanho do buraco
    }
});

--- TIPO 5: RADAR (Aranha) ---

Uso: Comparar múltiplas variáveis em diferentes dimensões
Ideal para: Análises multidimensionais, perfis

Exemplo:
new Chart(ctx, {
    type: 'radar',
    data: {
        labels: ['Qualidade', 'Preço', 'Suporte', 'Velocidade', 'Estabilidade'],
        datasets: [{
            label: 'Satisfação',
            data: [4.5, 4.0, 3.5, 4.8, 4.2],
            backgroundColor: 'rgba(59, 130, 246, 0.2)',
            borderColor: '#3B82F6',
            pointBackgroundColor: '#3B82F6'
        }]
    },
    options: {
        responsive: true,
        scales: {
            r: {
                beginAtZero: true,
                max: 5
            }
        }
    }
});

--- TIPO 6: POLAR AREA (Área Polar) ---

Uso: Combina características de PIE e RADAR
Ideal para: Magnitudes e proporções simultaneamente

Exemplo:
new Chart(ctx, {
    type: 'polarArea',
    data: {
        labels: ['São Paulo', 'Rio de Janeiro', 'Minas Gerais'],
        datasets: [{
            data: [150, 80, 60],
            backgroundColor: [
                'rgba(255, 99, 132, 0.5)',
                'rgba(54, 162, 235, 0.5)',
                'rgba(255, 206, 86, 0.5)'
            ]
        }]
    },
    options: {
        responsive: true
    }
});
