================================================================================
1. VISÃO GERAL DO SISTEMA
================================================================================

O Sistema de Leads é uma plataforma completa de CRM (Customer Relationship 
Management) que gerencia todo o ciclo de vida de clientes potenciais, desde 
a captação inicial até o pós-venda.

PRINCIPAIS FUNCIONALIDADES:
✓ Gestão completa de leads (CRUD)
✓ Classificação automática por temperatura (7 níveis)
✓ Separação por departamentos (Vendas e Pós-Venda)
✓ Rastreamento do funil de vendas (10 status)
✓ Gestão de pós-venda (11 estratégias)
✓ Avaliação de satisfação do cliente (5 níveis)
✓ Monitoramento financeiro
✓ Múltiplos canais de origem
✓ Sistema de pontuação (score 0-100)
✓ Integração com Facebook, WhatsApp, Telefone

USUÁRIOS DO SISTEMA:
- Equipe de Vendas
- Equipe de Pós-Venda
- Gerentes e Supervisores
- Administradores do Sistema

--- API 2: LEADATRIBUTES.PY (PORTA 5002) ---
Responsabilidade: Classificação automática e integrações

ENDPOINTS:

POST /api/leads/classify/:id_cliente
    Descrição: Classifica automaticamente uma lead baseado em contrato
    Lógica:
        - Sem contrato → Vendas (NovaLead)
        - Contrato ativo + satisfação ≥4 → Pós-Venda (Ativas Satisfeitas)
        - Contrato ativo + satisfação <4 → Pós-Venda (Ativas Insatisfeitas)
        - Contrato cancelado → Vendas (Recuperar)
    Retorna:
        {
            "success": true,
            "data": {
                "departamento": "vendas",
                "prioridade": "MuitoQuente",
                "status_vendas": "Recuperar",
                "score": 75
            }
        }

POST /api/leads/classify-all
    Descrição: Classifica todas as leads em lote
    Retorna:
        {
            "success": true,
            "classified": 145,
            "message": "145 leads classificadas"
        }

GET /api/leads/facebook-webhook
    Descrição: Verificação do webhook Facebook
    Query: hub.mode, hub.verify_token, hub.challenge
    Retorna: hub.challenge

POST /api/leads/facebook-webhook
    Descrição: Recebe leads do Facebook Lead Ads
    Body: Formato Facebook Webhook
    Cria lead automaticamente com origem="RedeFacebook"

POST /api/leads/from-contact
    Descrição: Cria lead de contato (WhatsApp/Telefone/Email)

================================================================================

FLUXO 1: CRIAR NOVA LEAD

Frontend → Backend:
    POST http://localhost:5001/api/leads
    Content-Type: application/json
    Body: {
        "razao": "Empresa Nova",
        "cnpj_cpf": "12345678901",
        "email": "contato@empresa.com",
        "fone": "1199998888",
        "cidade": "São Paulo",
        "uf": "SP",
        "prioridade": "NovaLead",
        "departamento": "vendas",
        "status_vendas": "NovaLeadVendas",
        "origem": "Telefone"
    }

Backend → Banco de Dados:
    INSERT INTO lead (
        razao, cnpj_cpf, email, fone, cidade, uf, 
        prioridade, departamento, status_vendas, origem,
        ativo, score, status_debito, data_criacao
    ) VALUES (
        'Empresa Nova', '12345678901', 'contato@empresa.com',
        '1199998888', 'São Paulo', 'SP',
        'NovaLead', 'vendas', 'NovaLeadVendas', 'Telefone',
        'S', 0, 'EmDia', NOW()
    )
    RETURNING id

Backend → Frontend:
    HTTP 201 Created
    {
        "success": true,
        "data": {
            "id": 123,
            "razao": "Empresa Nova",
            ... (todos os campos)
        },
        "message": "Lead criado com sucesso"
    }

Frontend: Exibe mensagem de sucesso, fecha modal, recarrega lista

FLUXO 2: EDITAR LEAD EXISTENTE

Frontend → Backend:
    1. GET http://localhost:5001/api/leads/123
       (Busca dados atuais para preencher formulário)
    
    2. PUT http://localhost:5001/api/leads/123
       Content-Type: application/json
       Body: { ...campos modificados }

Backend → Banco de Dados:
    UPDATE lead SET
        razao = 'Empresa Editada',
        email = 'novo@email.com',
        prioridade = 'Quente',
        data_atualizacao = NOW()
    WHERE id = 123

Backend → Frontend:
    HTTP 200 OK
    {
        "success": true,
        "message": "Lead atualizado com sucesso"
    }

FLUXO 3: CLASSIFICAÇÃO AUTOMÁTICA

Usuário clica em "Classificar" na interface

Frontend → Backend:
    POST http://localhost:5002/api/leads/classify/123

Backend → Banco de Dados:
    1. SELECT * FROM lead WHERE id = 123
    2. SELECT status FROM cliente_contrato WHERE id_cliente = 123
    3. SELECT nota FROM cliente_satisfacao WHERE id_cliente = 123
                                                   (se configurado)

Backend: Executa lógica de classificação:
    - Verifica se tem contrato
    - Verifica status do contrato (A=Ativo, C=Cancelado)
    - Verifica satisfação (se disponível)
    - Calcula score
    - Define departamento, prioridade, status

Backend → Banco de Dados:
    UPDATE lead SET
        departamento = 'pos_venda',
        prioridade = 'Quente',
        status_pos_venda = 'AtivasSatisfeitas',
        nivel_contentamento = 'Excelente',
        score = 85,
        data_ultima_classificacao = NOW()
    WHERE id = 123

Backend → Frontend:
    HTTP 200 OK
    {
        "success": true,
        "data": {
            "departamento": "pos_venda",
            "prioridade": "Quente",
            "status_pos_venda": "AtivasSatisfeitas",
            "nivel_contentamento": "Excelente",
            "score": 85
        }
    }

Frontend: Atualiza badges na tabela com novas cores

7. SISTEMA DE CLASSIFICAÇÃO
================================================================================

NÍVEIS DE PRIORIDADE (TEMPERATURA):

1. NovaLead (🆕 Verde claro)
   - Lead recém-cadastrada
   - Sem interação ainda
   - Score padrão: 0

2. Ruim (⚫ Cinza)
   - Lead sem potencial identificado
   - Baixo interesse
   - Score típico: 0-20

3. Fria (❄️ Azul claro)
   - Lead com baixo interesse
   - Necessita aquecimento
   - Score típico: 20-35

4. Morna (🌡️ Laranja claro)
   - Lead com interesse moderado
   - Em processo de qualificação
   - Score típico: 35-50

5. Quente (🔥 Laranja)
   - Lead com alto interesse
   - Próxima de fechar negócio
   - Score típico: 50-70

6. MuitoQuente (🔥🔥 Laranja escuro)
   - Lead com altíssimo interesse
   - Demanda atenção prioritária
   - Score típico: 70-85

7. Queimando (🔥🔥🔥 Vermelho)
   - Lead urgente
   - Requer ação imediata
   - Risco de perder se não agir
   - Score típico: 85-100

STATUS DE VENDAS (FUNIL):

NovaLeadVendas → Contato → Qualificacao → Proposta → Negociacao 
→ AguardandoAssinatura → GanhaSucesso

Saídas laterais: Perdida, SemInteresse, Recuperar

1. NovaLeadVendas: Lead no setor de vendas aguardando primeiro contato
2. Contato: Primeiro contato realizado
3. Qualificacao: Lead sendo qualificada (BANT: Budget, Authority, Need, Timeline)
4. Proposta: Proposta comercial enviada
5. Negociacao: Em negociação de valores/condições
6. AguardandoAssinatura: Aguardando assinatura de contrato
7. GanhaSucesso: Venda concretizada (vira cliente)
8. Perdida: Oportunidade perdida
9. Recuperar: Cliente inativo a ser recuperado
10. SemInteresse: Lead desinteressada (não insistir)

STATUS PÓS-VENDA (ESTRATÉGIAS):

1. AcompanhamentoPersonalizado: Cliente VIP com atendimento dedicado
2. TreinamentoSuporte: Necessita treinamento sobre produto/serviço
3. ProgramasDeFidelidade: Participante de programa de fidelidade
4. AcoesDeAgradecimento: Ações de reconhecimento e gratidão
5. AnaliseDeUsoEFeedback: Análise de uso e coleta de feedback
6. AtivasSatisfeitas: Clientes ativos e satisfeitos (manutenção)
7. AtivasInsatisfeitas: Clientes ativos mas insatisfeitos (prioridade alta)
8. RecuperarSatisfacao: Ação para recuperar satisfação
9. UpsellCrossSell: Oferecer produtos/serviços adicionais
10. AtendimentoRapido: Prioridade no atendimento
11. RelacionamentoDeLongoPrazo: Foco em relacionamento duradouro

NÍVEIS DE CONTENTAMENTO:

⭐⭐⭐⭐⭐ Excelente (5/5): Cliente extremamente satisfeito
⭐⭐⭐⭐ Bom (4/5): Cliente satisfeito
⭐⭐⭐ Regular (3/5): Cliente neutro
⭐⭐ Ruim (2/5): Cliente insatisfeito
⭐ Péssimo (1/5): Cliente muito insatisfeito (churn risk)

STATUS FINANCEIRO:

✅ EmDia: Pagamentos em dia
⚠️ EmAtraso: Pagamentos atrasados (risco de churn)

CANAIS DE ORIGEM:

📄 Panfleto: Divulgação impressa
📞 Telefone: Ligação ativa ou receptiva
📧 Email: E-mail marketing
💬 WhatsApp: Mensagem WhatsApp
📷 Instagram: Rede social Instagram
📘 Facebook: Rede social Facebook (incluindo Lead Ads)
🔍 Google: Google Ads ou busca orgânica
🐦 Twitter: Rede social Twitter
🏪 Loja: Visita presencial

SISTEMA DE PONTUAÇÃO (SCORE 0-100):

Cálculo do score baseado em:
- Status do contrato (+40 pontos se ativo)
- Satisfação do cliente (+20 pontos se excelente)
- Status financeiro (+15 pontos se em dia)
- Tempo de relacionamento (+10 pontos se > 12 meses)
- Engajamento (+15 pontos se alta interação)

Faixas de score:
- 0-29: Baixo (vermelho)
- 30-59: Médio (amarelo)
- 60-100: Alto (verde)

LÓGICA DE CLASSIFICAÇÃO AUTOMÁTICA:

IF sem_contrato:
    departamento = "vendas"
    prioridade = "NovaLead"
    status_vendas = "NovaLeadVendas"

ELIF contrato_ativo:
    departamento = "pos_venda"
    
    IF satisfacao >= 4:
        status_pos_venda = "AtivasSatisfeitas"
        prioridade = "Quente"
        nivel_contentamento = "Excelente" ou "Bom"
    ELSE:
        status_pos_venda = "AtivasInsatisfeitas"
        prioridade = "MuitoQuente"  # Alta prioridade!
        nivel_contentamento = "Regular", "Ruim" ou "Pessimo"

ELIF contrato_cancelado:
    departamento = "vendas"
    prioridade = "MuitoQuente"  # Recuperar é prioritário
    status_vendas = "Recuperar"
    
    IF satisfacao_anterior >= 4:
        nivel_contentamento = "Bom"  # Cliente estava satisfeito
    ELSE:
        nivel_contentamento = "Ruim"  # Cliente insatisfeito
